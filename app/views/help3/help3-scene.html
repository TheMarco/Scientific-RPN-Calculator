Unlike algebraic calculators, this application uses a logic called "Reverse Polish Notation" (RPN). While Algebraic calculators place operators between the numbers (for example 3 + 5), RPN places them after the numbers.

With RPN we can solve complicated calculations without the need of parenthesis or punctuation. This is possible because the calculator automatically retains and returns intermediate results. The magic behind this is the automatic memory stack and the key. The amount of keystrokes needed to perform operations is usually much smaller on an RPN calculator than it is on an algebraic one. While it takes a little while to get used to solving problems the RPN way most people won't want to go back to the algebraic way after getting to know RPN.

The Stack

This calculator features a 5-level numeric stack in which intermediate results are automatically stored. The stack works on a last-in, first-out basis. Look at this as a pile of office paperwork on which new papers are placed at the bottom of the pile as more work comes in. Now let's assume we process the paperwork by always removing the paper on the bottom of the pile and deal with it. This means that older paperwork will stay on the pile unless no new paperwork is brought in for a longer period of time. As a last rule, let's assume that if the pile becomes higher than 5 items we start feeling overworked and just silently discard the item on top of the pile. No one is gonna need that anymore anyway since it's been sitting there for such a long time!

The number visible in the calculator display is the X register. Numbers that are keyed in or numbers that are the result of an operator will displayed and placed in the X register. Using the to terminate digit entry will cause the stack to lift (a new piece of paperwork is placed on the bottom of the pile).

The value on top of the stack (b) has been copied and is now placed in both the A and the B register. This behavior can come in quite handy with certain calculations.

LASTx

The calculator has a special memory register which contains the last value that was in the X register before a numeric operation was performed. This is called the LASTx register. Pushing will recall this value and place it in the X register while lifting the stack.

UNDO

In addition to the LASTx register, the calculator keeps a copy of the entire stack at all times. When performing any operation that changes the stack it will keep the values representing the stack before the operation was performed. Pressing will revert the stack to what it was before performing an operation.

Storage Register Operations

Your calculator has 10 storage registers that can hold any value you want it to remember for later use. The registers are named R0 to R9. Values stored in these registers will persist between sessions with the calculator, even if the phone has been switched off inbetween sessions.

IMPORTANT: R2 to R7 are also used by the statistics functions which are discussed in another chapter of this guide. If you use statistics operations during your calculations make sure you only use R0, R1, R8 and R9.

To store a value in a register use followed by to . The value in the display at the moment of keying in this sequence will be stored in the register.

To clear a register, store the value 0 in that register.
